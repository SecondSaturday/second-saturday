# Task: Configure Deployment and CI/CD

## Description
Set up Vercel for web deployment and GitHub Actions for continuous integration. Configure automatic deployments, preview environments, and PR checks.

## Acceptance Criteria
- [ ] GitHub repository created (or connected)
- [ ] Vercel account created and linked to GitHub
- [ ] Environment variables configured in Vercel
- [ ] Production deployment on main branch
- [ ] Preview deployments on pull requests
- [ ] GitHub Actions workflow for CI (lint, type-check, test)
- [ ] PR template created
- [ ] Failed CI blocks merge to main
- [ ] Build completes in under 5 minutes

## Technical Details
- Connect repo to Vercel via GitHub integration
- Create .github/workflows/ci.yml with jobs for lint, type-check, test
- Configure branch protection rules on main
- Create .github/PULL_REQUEST_TEMPLATE.md
- Set Vercel build command and output directory
- Configure all environment variables in Vercel dashboard

## Dependencies
- [ ] Task 001 completed (project structure exists)
- [ ] GitHub account with repository access
- [ ] Vercel account

## Effort Estimate
- Size: M
- Hours: 4-6
- Parallel: true (can run alongside 003-006)

## Definition of Done
- [ ] Push to main triggers auto-deploy
- [ ] Production URL accessible
- [ ] Preview deploys work for PRs
- [ ] CI runs on every PR
- [ ] Failed CI blocks merge
