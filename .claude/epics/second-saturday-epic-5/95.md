# Task: Create React Email Templates

## Description
Build two React Email templates using `@react-email/components` (already installed): the main newsletter email and the missed-month email. These are the first React Email templates in the project.

## Acceptance Criteria
- [ ] `NewsletterEmail.tsx` template created with React Email components
- [ ] Newsletter email is responsive (600px max width)
- [ ] Uses inline CSS only (no external stylesheets) for Gmail/Outlook compatibility
- [ ] Newsletter organized by prompt with member names as headers
- [ ] Photos displayed inline (compressed, max 1200px wide)
- [ ] Video thumbnails with play button overlay linking to web view URL
- [ ] "View in App" CTA button linking to newsletter web view
- [ ] Footer with unsubscribe link
- [ ] Circle header: icon, name, issue number, date
- [ ] `MissedMonthEmail.tsx` template created (encouraging message, sad puppy gif)
- [ ] Both templates work without images (graceful text fallback)
- [ ] Email total size target <5MB

## Technical Details
- Create templates in `src/emails/` directory (new directory, standard React Email convention)
- Use `@react-email/components`: `Html`, `Head`, `Body`, `Container`, `Section`, `Row`, `Column`, `Text`, `Img`, `Link`, `Button`, `Hr`, `Preview`
- Props interface for `NewsletterEmail`: `{ circleIcon, circleName, issueNumber, date, sections: Array<{ promptTitle, responses: Array<{ memberName, text, media }> }>, viewInAppUrl, unsubscribeUrl }`
- Props interface for `MissedMonthEmail`: `{ circleName, circleIcon, nextDeadline, viewCircleUrl }`
- Follow React Email best practices: table-based layout for Outlook, inline styles, no flexbox/grid
- Test rendering with `npx react-email dev` if available

## Dependencies
- [ ] None - templates are self-contained React components
