# Task: Build NotificationPreferences UI screen

## Description
Create a notification preferences screen accessible from user settings. Provides global toggles for submission reminders and newsletter-ready notifications. Changes persist immediately to Convex.

## Acceptance Criteria
- [ ] Notification preferences accessible from user settings page
- [ ] Toggle for submission reminders (on/off, default on)
- [ ] Toggle for newsletter ready notifications (on/off, default on)
- [ ] Changes saved immediately to Convex (no save button needed)
- [ ] Loading state while preferences load
- [ ] Toast confirmation when preferences updated
- [ ] Settings sync across devices (via Convex reactivity)

## Technical Details
- Create `src/components/NotificationPreferences.tsx`
- Add link/section in existing settings page (`src/app/dashboard/settings/page.tsx`)
- Use `useQuery` for `getNotificationPreferences` and `useMutation` for `updateNotificationPreferences`
- Follow existing UI patterns from `CircleSettings.tsx` (dialog/card style, toggle switches)
- Use shadcn Switch component for toggles
- Use sonner toast for feedback
